Human Computer Interaction

In this section, we will show the initial research, sketches and protopyes that we came up with for the HCI section of our project. We will also give an overview of our final User Interface.

HCI Related Research

Below is a list of all the research and considerations that took into account when designing our solution.

Positioning of information on the glasses have varied effects depending on the type of task being performed [1]

Central positioning Peripheral positioning
If the user is performing an intensive primary task, placing data in the centre of their view allows for maximum continued focus on that particular task while also being able to interpret the information displayed. This option allows for more comfort on the wearer as they won’t have a high concentration of information in particular area (including both information from the glasses as well as information from the real world).

Other AR considerations:

Field Of View (FOV) Field of view is important as you don’t want information to be obstructive but also be equally as readable so that the experience feels natural to the user as well as the people around the user - this is a critical consideration as it is a key requirement.
Interaction with virtual information The method in which the user interacts with the glasses is an important factor both for the user and the requirements of the project. The interaction method should be intuitive and easy to execute especially for less technically advanced users.The interaction methods should not affect the flow of the consultation i.e. they should feel natural. We have found the main feasible methods are external buttons to the glasses, buttons on the glasses frame and an android application to interact with the glasses.
Physical Ergonomics There are a variety of smart glasses available with varying feature sets. If we look at the market as well as the specifications the two best suited products are the Google glasses and Vuzix glasses. The main advantages being natural looks, affordability, sufficient interactive interfaces and comfort.

It’s important to consider conventional screen display methods since they are similar in some regards but caution needs to be taken. For instance, information must be prioritised to not overload the user and maximise retention.


HCI principles to be applied in AR and specifically to our project [2][3]:

  • When adding elements → consider usefulness of the feature in relation to the goal of the application
  • Reduce cognitive load → Do not overwhelm the user with information
  • Reduce physical activity required to interact with the virtual environment
  • Reduce the timeand steps it takes to complete a task
  • Responsiveness → Make the application work as fast as possible to give a good user experience
  • Error Tolerance → Reduce the amount of bugs in order to avoid user frustration

Initial application diagram

To start with, we decided to sketh the application user interface and control flow using wireframes. These guided the initial development but we have made lots of changes in order to produce our final application.

Initial User Interface

From our specification and requirements, we built a user interface prototype with placeholders for the various pages(Image on the left) that we thought we will need to implement our functionality. We also started on the main interface page for sessions (Image on the right). We put as few buttons as possible as we are always aiming for simplicity and ease of use.

Final User Interface

For our final iteration in this part of the project, we made some important changes to our User Interface. Firstly, we added most of the pages from the drawer. Some of them still don't have any functionality but they were not features that were in our Requirements list. We decided to keep them as we think that they would be good additions in the future. We also made some important changes to the main session UI. The buttons are now more expressive and we also added a button to end the session (Back button) as well as one to check all words that have been said during the session.
