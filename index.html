Learn about the design and implementation of our solution

As a patient in a GP consultation, it can be very distracting to have a medical student taking notes or looking up definitions on the side. Therefore GPs often tell trainees not to take notes during consultations, which hinders their learning since it is hard to remember all the terms that you donâ€™t understand. We have developed a smart glasses application that aims to solve those two problems. Our solution will allow trainees to display medical word definitions in real time and come back to them for further studying, all whilst minimising distraction during the consultation.

Key Features

Incorporated Medical Dictionary

We are using an external Dictionary API to search for defintitions of words. By having a specific medical dictionary, we can have more accurate definitions.

Interpretable Logs

When the user choses to display a word definition, it is saved in a local file so that the user can come back to words. Additionaly, by saving the data in XML format, we facilitate its potential usage in machine learning.

Real Time Display

Words are picked up in real time by our speech to text functionality and the user has full control of how long they wish to display a word and its definition. No hard coded timers, fully personalised.

Expandability

We have used a specific dictionary in this project, but the possibilities are endless for future development. Replace the API with any other dictionary and voila! You have found a new use for the application. Glasses are also optional, for situations that only require a phone!
